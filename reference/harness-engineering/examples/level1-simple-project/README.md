# Level 1: 简单任务管理器 - Harness 工程示例

这是 Harness 工程入门级示例项目，展示了如何在简单的 Python CLI 项目中应用 Harness 工程原则。

## 项目特点

- ✅ 完整的 CLAUDE.md 指南
- ✅ 类型注解和文档字符串
- ✅ 自动化代码检查（Black、Ruff、MyPy）
- ✅ 完整的单元测试
- ✅ 清晰的错误处理

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 运行程序

```bash
python src/main.py
```

### 3. 运行测试

```bash
pytest tests/ -v
```

### 4. 代码检查

```bash
# 类型检查
mypy src/

# 代码格式化
black src/ tests/

# 代码风格检查
ruff check src/ tests/
```

## Harness 工程要点

### 1. CLAUDE.md 的作用

CLAUDE.md 是这个项目的"Harness"核心：
- 告诉 Claude 项目的基本信息
- 定义代码规范（命名、类型注解、文档）
- 列出常用命令
- 强调重要规则（错误处理、输入验证）

### 2. 自动化工具链

- **Black**: 自动格式化代码
- **Ruff**: 检查代码风格和文档
- **MyPy**: 类型检查
- **pytest**: 运行测试

这些工具确保代码质量，减少人工审查负担。

### 3. 测试覆盖

- 单元测试覆盖所有公共方法
- 边界情况测试
- 持久化测试
