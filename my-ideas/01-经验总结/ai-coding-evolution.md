# AI 编程经验演进记录

> 本文档汇总了在使用 AI 编程工具过程中的思路、感受和实践经验

---

## 一、核心思路

### 1.1 多任务并行

- **Git Worktree**
  - Claude Code 的 worktree 最简单
  - vibe-kanban 比较新颖，对开发比较友好

### 1.2 上下文控制

- 子代理（Sub Agent）
- Agent Teams

### 1.3 项目记忆积累

- 自己研发的复盘命令 `/retrospect`
- 增加丰富项目记忆

---

## 二、工具组合实践

### 2.1 核心工具链

1. **Claude Code + cc-switch 工具组合**

2. **playwright-mcp 工具**
   - 适用场景：编写用户手册、某个系统的调研

### 2.2 项目知识组织

- 积累过程：每遇到一个问题，项目中积累一条记录
- `CLAUDE.md` 保持精简
- `rules/` 目录中累加规则

---

## 三、设计-实现-测试闭环

### 3.1 设计阶段

- 使用 Lingma IDE 写设计 Markdown

### 3.2 实现阶段

对于比较长的任务：
1. 使用 Sub Agent
2. 使用 Agent Team
3. 先把详细的计划文件输出到 Markdown，审阅后再启动多个 Agent 协作编码
4. 使用 Git Worktree 并行开发

### 3.3 测试阶段

1. **单元测试**（类库等）
   - 注意：单元测试有维护成本

2. **Karate 测试**
   - 在开发业务接口的场景中比较好用

---

## 四、关键感受

### 4.1 PRD 质量决定成败

> 原型（/PRD）的质量高，AI 编程就事半功倍

**反之，如果 PRD 本身模糊不清、甚至自相矛盾：**
- AI 编码需要先细化设计、二次设计等
- 大概率会越走越偏、飞速跑偏
- 没法验收，无法判断对错

### 4.2 奢侈但有效的策略

如果尝试了很多次和 AI 交互，没有得到好的方案，怎么办？

**答案：使用 Opus 4.6**
