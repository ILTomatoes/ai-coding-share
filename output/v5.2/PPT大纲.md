  - 后果：已部署环境出现文件摘要不一致错误
  - 复盘改进：Flyway 已冻结脚本严禁修改，所有变更只走新建补丁文件
- 卡片3：**权限过滤理解偏差**
  - 场景：优化权限相关代码
  - 问题：AI 对"前端可见性控制权限"策略理解不深，把应交给前端的逻辑保留在后端
  - 后果：后端多了不必要的权限检查
  - 复盘改进：所有用户级权限判断（包括"不能操作自己提交的"）都应交给前端

### P23 什么是 SDD（规范驱动开发）
- **排版**：定义 + Vibe Coding 对照 + Spec 契约 + 收益
- **定义**：SDD = Specification-Driven Development（规范驱动开发）
- 一句话：**先写说明书，再让 AI 写代码**的开发新范式
- Spec 是人与 AI 之间的**契约**，是**唯一真理来源**
- 代码只是 Spec 的"**派生物**"
- 对比（Vibe Coding vs SDD）：
  - ❌ 氛围编程：凭感觉提需求 → AI 猜着写 → 写错花几小时修
  - ✅ SDD：规范第一，代码第二——拒绝瞎猜
- 类比：Spec 是"建筑图纸"，AI 是"施工队"
  - 没有图纸 = 施工队自由发挥（不可控）
  - 有图纸 = 按图施工（可预测）
- Spec 包含四要素：
  - 目标与价值 | 上下文与约束 | 功能逻辑 | 验证标准（自带 TDD 基因）

### P24 SDD 四阶段工作流
- **排版**：横向流程图（四节点） + 右侧三个收益卡片
- 四阶段流程图（箭头连接）：
  - 📝 **Specify（定义）**：写清楚"做什么"和"为什么"
  - 🏗️ **Plan（计划）**：确定技术架构和实现路径
  - ✂️ **Task（任务）**：拆解为 AI 可执行、可验证的原子任务
  - 🤖 **Implement（实现）**：AI 按图纸干活，人审核验证
- 三个核心收益（右侧卡片）：
  - 🔮 **可预测性**：动手前就知道改哪些文件、怎么改，不是"开盲盒"
  - ✅ **高质量**：Spec 强制包含测试标准 → 代码自带 TDD 基因
  - 📖 **易维护**：逻辑为什么存在，答案在 Spec 里
  - 💰 **ROI 答疑**：小改动跳过Spec；超过一天的功能完整Spec流程；判断标准——给新同事讲需超过5分钟就值得写
- 底部金句（大字，亮蓝）：从"码农"变成"系统架构师"——你画图纸，AI 盖楼

### P30 什么是 Harness Engineering
- **排版**：核心公式 + 定义 + 两大控制机制 + 核心原则 + 一句话总结
- **核心公式**（大字突出）：Agent = Model + Harness（LangChain 提出）
  - 模型是马，Harness 是缰绳——光有马不行，还需要一整套驾驭系统
- **Harness = 给 AI 烈马套上缰绳的马具**
  - AI 速度快、力量大，但不套上缰绳就会跑偏
  - Harness 把这股力量引导到你需要的方向上
- 两大控制机制（形成闭环）：
  - 🛡️ **Guides（前馈控制）**：行动前设好规则，让它沿正确方向走
    - 📋 CLAUDE.md、代码规范、架构决策记录、权限分级
    - 类比：高速公路护栏——不需要每秒纠正，护栏在就行
  - ✅ **Sensors（反馈控制）**：做完后检测对不对
    - 自动化测试、代码 lint、CI 流水线
  - **闭环逻辑**：好的 Harness = Guides + Sensors，防患于未然 + 亡羊补牢
- 三层上下文供给：📋 项目级（CLAUDE.md、rules/）→ 📝 任务级（Plan、Spec）→ 💬 会话级（对话引导）
- **核心原则：约束先行** —— 规则不是靠口头约定，靠自动化来强制执行
- 目标：**让 AI 在你的项目里工作得又快又准**

  - 团队成员共享配置

### P41 技巧一：小步快跑，持续验证
- **排版**：流程循环图
- **设计 → 实现 → 测试**，每完成一个模块就验证
- AI 生成快 → 错误积累也快 → 一次贪多反而更慢
- 遇到问题再"磨刀"，不提前过度设计
- 实际教训：aip-portal 早期一次做大模块导致批量返工

### P42 技巧二：先建立认知，再让 AI 干活
- **排版**：对比图（2小时 vs 16天）
- 核心原则：
  - **你的认知边界 = AI 效率上限**
  - 越不了解的领域，越要先花时间调研
- A2A 教训：**2 小时调研 vs 16天返工**
- 实践建议：
  - 新领域：先让 AI 帮你调研、梳理认知框架，再进入编码
  - 熟悉领域：可以直接进入设计-实现
